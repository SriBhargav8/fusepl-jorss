# First Unicorn Startup — Unicorn Score Valuation Platform

## Overview

**Domain:** firstunicornstartup.com

**Goal:** Build India's definitive startup valuation tool that gives founders an instant "Unicorn Score" (0-100) plus a professional-grade valuation breakdown — free as a lead magnet, with a certified report upsell at Rs 14,999.

**Architecture:** Next.js 14 client-side application with Supabase backend for data capture. All valuation computation runs in the browser (zero API cost). Damodaran India benchmarks auto-populate industry-specific inputs. AI narrative analysis via Claude API adds qualitative insight to the unlocked report.

**Tech Stack:** Next.js 14 (App Router), Tailwind CSS, shadcn/ui, Zustand, Recharts, jsPDF, Supabase (PostgreSQL + Auth + Edge Functions), Claude API (for AI narrative), Vercel deployment.

## Unicorn Score Algorithm

### Score Composition (0-100):

```
UNICORN SCORE = Weighted Average of 5 Components:

1. VALUATION STRENGTH (40% weight)
   - Run 5 valuation methods:
     a) Scorecard Method (Bill Payne) — 7 factors weighted to 100%
     b) Berkus Method — 5 factors × Rs 0-4 Cr each, max Rs 20 Cr
     c) Revenue Multiple — sector-specific multiples from Damodaran India
     d) Risk Factor Summation — 12 risk dimensions rated -2 to +2
     e) DCF — 5-year projections with Damodaran India WACC
   - Each method produces: valuation estimate + confidence score (0-1)
   - Weighted average by confidence = primary valuation
   - Score component = percentile rank vs sector/stage benchmark table

2. GROWTH TRAJECTORY (20% weight)
   - Revenue growth rate vs Damodaran India sector median
   - Burn rate efficiency: revenue / monthly burn ratio
   - Runway health: months remaining (>18 = excellent, <6 = critical)
   - Unit economics: LTV/CAC ratio health (>3x = excellent)

3. MARKET POSITION (15% weight)
   - TAM reasonableness check (vs industry data)
   - SAM/SOM as % of TAM (sanity check)
   - Competition level adjustment
   - Barrier to entry / moat strength

4. TEAM QUALITY (15% weight)
   - Founder experience score (1-5)
   - Domain expertise (1-5)
   - Previous exits bonus
   - Team completeness (CTO + CMO + CFO coverage)

5. PRODUCT READINESS (10% weight)
   - Development stage: idea (20) → MVP (40) → beta (60) → production (80) → scaling (100)
   - Product-market fit signals
   - Tech differentiation score (1-5)
   - IP / patent bonus
```

### Probabilistic Valuation Range (Monte Carlo):

```
For each of 10,000 iterations:
  1. Sample growth rate from Normal(stated_growth, stated_growth × 0.3)
  2. Sample margin from Normal(stated_margin, stated_margin × 0.2)
     → Mean-reverts toward Damodaran India sector median over 5 years
  3. Discount rate = Damodaran India WACC for sector ± random(0, 0.02)
  4. Run DCF with sampled parameters
  5. Store result

Output:
  P10 (conservative) = 10th percentile of 10,000 results
  P50 (base case)    = 50th percentile
  P90 (optimistic)   = 90th percentile

Display as: "Rs 6 Cr (conservative) — Rs 10 Cr (base) — Rs 16 Cr (optimistic)"
```

### Damodaran India Auto-Population:

When founder selects a sector, the following are auto-populated silently:
- **Beta** → from betaIndia.xls (mapped to closest Damodaran industry)
- **WACC** → from waccIndia.xls
- **Revenue multiple** → from psIndia.xls (EV/Revenue median)
- **EV/EBITDA** → from vebitdaIndia.xls
- **Operating margin** → from marginIndia.xls (sector median as mean-reversion target)

## Valuation Methods — Specifications

### Method 1: Scorecard (Bill Payne)

**Inputs:** 7 factor scores (from wizard steps 2-6)
**Weights:**
- Management team: 30%
- Market opportunity: 25%
- Product/technology: 15%
- Competition: 10%
- Sales/marketing: 10%
- Need for funding: 5%
- Other factors: 5%

**Calculation:**
1. Get typical pre-money valuation for stage from benchmark table
2. Score each factor as 0% to 150% of its weight
3. Sum adjustments → apply to base valuation
4. Confidence: 0.7 for pre-revenue, 0.5 for idea stage

### Method 2: Berkus

**Inputs:** 5 factor assessments (from wizard)
**Factors (each Rs 0-4 Cr):**
1. Sound idea / business model strength
2. Working prototype / product stage
3. Quality management team
4. Strategic relationships
5. Product rollout / early sales traction

**Calculation:**
- Sum of 5 factors, capped at Rs 20 Cr
- Confidence: 0.8 for pre-seed, 0.6 for seed+

### Method 3: Revenue Multiple

**Inputs:** Annual revenue, sector, growth rate
**Calculation:**
1. Base multiple from Damodaran India EV/Revenue for sector
2. Growth adjustment: +2x if >200% growth, +1x if >100%
3. SaaS model premium: +1.5x
4. Unit economics bonus: +1x if LTV/CAC > 5
5. Final = Revenue × adjusted multiple
6. Confidence: 0.9 if revenue > Rs 1 Cr, 0.5 if < Rs 10L
7. Not applicable if pre-revenue (excluded from scoring)

### Method 4: Risk Factor Summation

**Inputs:** 12 risk dimensions (auto-assessed from wizard data)
**Dimensions (each rated -2 to +2):**
1. Management risk
2. Stage of business
3. Legislation/regulatory risk
4. Manufacturing/supply risk
5. Sales and marketing risk
6. Funding/capital risk
7. Competition risk
8. Technology risk
9. Litigation risk
10. International risk
11. Reputation risk
12. Potential exit

**Calculation:**
1. Per-factor adjustment: Rs 25L (pre-seed) to Rs 5 Cr (Series A+)
2. Sum adjustments → apply to stage-based starting valuation
3. Confidence: 0.75

### Method 5: DCF (Discounted Cash Flow)

**Inputs:** Revenue, growth, gross margin, operating margin, WACC (from Damodaran)
**Calculation:**
1. Project revenue for 5 years using stated growth with 0.85/year decay
2. Compute FCF = Revenue × operating margin × (1 - tax rate 25%)
3. Discount at Damodaran India WACC for sector
4. Terminal value = FCF_year5 × (1 + India_GDP_growth) / (WACC - India_GDP_growth)
   - India_GDP_growth capped at 5.5%
5. Enterprise value = PV of FCFs + PV of terminal value
6. Confidence: 0.85 if revenue > Rs 5 Cr, 0.4 if pre-revenue

### Final Valuation:
- Weighted average of applicable methods (weighted by confidence)
- Methods with confidence < 0.3 are excluded
- Display individual method results + weighted composite

## Cap Table Simulator

**Inputs (from wizard + additional form in report):**
- Current shares outstanding
- ESOP pool %
- Investment amount being raised
- Pre-money valuation (from Unicorn Score result)

**Outputs:**
- Post-money valuation
- Investor ownership %
- Founder dilution %
- ESOP dilution %
- Waterfall distribution at 3 exit scenarios (1x, 3x, 5x of post-money)

**ESOP Recommendations by Stage:**
- Idea/MVP: 10-12%
- Pre-seed/Seed: 12-15%
- Series A: 12-20%
- Series B+: 8-15%

## Investor Matching

**Database:** 25+ Indian VCs with:
- Name, type (VC/angel/accelerator/corporate-vc)
- Check size range (min/max in INR)
- Stage preference
- Sector focus
- Notable investments
- Sweet spot description

**Matching Algorithm:**
1. Filter by sector match
2. Filter by stage match
3. Filter by check size (raise amount within investor's range)
4. Score remaining by fit (sector specialist > generalist)
5. Return top 10 ranked matches with reasoning

**Display:** "Based on your profile, these investors are the best fit:"
- Investor name + type + check size
- Why they match (sector focus, stage alignment)
- Notable portfolio companies in your sector

**Provider:** Claude API (claude-sonnet-4-6)

**Trigger:** Runs server-side (Supabase Edge Function) after email is captured. Results cached and displayed in the unlocked report.

**Prompt Structure:**
```

Revenue: Rs {revenue} Cr, Growth: {growth}%

Market: TAM Rs {tam} Cr, SAM Rs {sam} Cr
Unicorn Score: {score}/100

Provide a 3-paragraph analysis:
1. What stands out (positive)
2. Key risks and concerns
3. What to do before your next fundraise
```

**Cost Control:**
- Only runs when email is captured (not for every wizard completion)
- Use claude-haiku-4-5 for cost efficiency (~$0.002 per analysis)

- Rate limit: max 100 AI calls/day initially

### Database Schema (Supabase PostgreSQL):

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT UNIQUE NOT NULL,
  full_name TEXT,
  company_name TEXT,
  created_at TIMESTAMPTZ DEFAULT now(),
  last_active TIMESTAMPTZ DEFAULT now(),
  source TEXT DEFAULT 'organic',
  referral_code TEXT UNIQUE DEFAULT encode(gen_random_bytes(4), 'hex')
);

CREATE TABLE valuations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT now(),

  -- Company inputs
  company_name TEXT,
  sector TEXT NOT NULL,
  stage TEXT NOT NULL,
  business_model TEXT,
  city TEXT,
  founding_year INT,
  team_size INT,

  -- Financial inputs
  annual_revenue DECIMAL,
  revenue_growth_pct DECIMAL,
  gross_margin_pct DECIMAL,
  monthly_burn DECIMAL,
  cash_in_bank DECIMAL,
  runway_months INT,
  cac DECIMAL,
  ltv DECIMAL,

  -- Market inputs
  tam DECIMAL,
  sam DECIMAL,
  som DECIMAL,

  -- Qualitative inputs (JSONB)
  team_assessment JSONB,
  product_assessment JSONB,
  market_position JSONB,

  -- Computed results
  unicorn_score INT,
  valuation_low DECIMAL,
  valuation_mid DECIMAL,
  valuation_high DECIMAL,
  method_results JSONB,
  damodaran_inputs JSONB,
  ai_narrative TEXT,

  version INT DEFAULT 1
);

CREATE TABLE certified_requests (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  valuation_id UUID REFERENCES valuations(id),
  user_id UUID REFERENCES users(id),
  status TEXT DEFAULT 'requested',
  payment_id TEXT,
  amount DECIMAL,
  report_type TEXT,
  requested_at TIMESTAMPTZ DEFAULT now(),
  delivered_at TIMESTAMPTZ
);

CREATE TABLE page_events (
  id BIGSERIAL PRIMARY KEY,
  session_id TEXT,
  event TEXT NOT NULL,
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_valuations_sector ON valuations(sector);
CREATE INDEX idx_valuations_stage ON valuations(stage);
CREATE INDEX idx_valuations_score ON valuations(unicorn_score);
CREATE INDEX idx_events_event ON page_events(event);
```
